# Architecture

Docket is a personal job application tracker: React SPA on Cloudflare Pages talking to a Hono API on Cloudflare Workers, backed by Neon Postgres (Hyperdrive) and R2 for documents.

## Components

| Piece | Tech | Host |
|---|---|---|
| Web | React 19 + Vite + Tailwind v4 + shadcn/ui | Cloudflare Pages |
| API | Hono + Drizzle (fixed OWNER_ID, no login) | Cloudflare Workers |
| DB | Neon Postgres (staging/main branches) | via Hyperdrive |
| Files | R2 buckets (presigned URLs) | Cloudflare R2 |

## Request flow

1. Browser loads SPA from Pages custom domain.
2. API routes use a fixed `OWNER_ID` and open a Drizzle client over Hyperdrive with that id mapped into `auth.user_id()` for RLS.
3. Document upload/download uses short-lived R2 signed URLs generated by the Worker.

## Monorepo

- `apps/web` — SPA
- `apps/api` — Worker
- `packages/shared` — Zod schemas and shared types

## Out of scope (v1)

- Resend reminder digests (documented for later)
- Multi-user collaboration UI (RLS is scaffolded for it)
